iCook Web APP

We used Figma to design iCook. We created a Lo-fi and a Hi-fi design. I learned that the more time and effort put into the design process makes it easier later on. We ran into a few problems along the way where a member would decide on their own on a decision because we didn't explicitly put it on our design which led to a not uniform app.

Part of our brainstorming process, we created user personas to understand the problems and needs of users. I also researched multiple food recipe manager APPs to find what makes up a recipe manager and the good and bad features so we can improve on it. We then came up with a list of features we wanted and narrowed it down to create a functional APP in less than 8 weeks. We did a fat marker sketch on Miro before we used Figma to create a more detailed layout. We then created UML diagrams and a quick sketch of the flow. Lastly, we made a roadmap of what each role should be doing and setting deadlines for ourselves. This was all done using Miro.

While working with a team of 10, I was in charge of back-end tasks. I learned how to fetch API data and populate into components we made. I made the buttons functional and contributed in storing data. Some features of iCook is that it highlights the step the user is currently on so they can easily find their spot. Since we targeted beginner chefs, we added images for ingredients and equipments. To cater to different user's preferences, they can filter their searches by cusisine, dietery needs or duration, for example. The users can also favorite recipes, create lists and manipulate their favorite lists.
